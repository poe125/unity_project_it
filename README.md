# \# Monster Summoner

# 

# Monster Summoner は、物理カードを読み取って遊ぶ AR（拡張現実）ターン制カードバトルゲームです。本 README では、ゲーム概要、ルール、技術構成、カードデータ仕様について説明します。

# 

# ---

# 

# \## 概要

# Monster Summoner は、スマートフォン（Android）を使用し、AR 空間にキャラクターや攻撃演出を表示しながら遊ぶカードバトルゲームです。

# 

# プレイヤーは物理カード（キャラ / 攻撃 / バフ）を読み取ることで、対応する 3D モデルやステータスを AR 上に召喚できます。

# 

# \### カードの種類

# \- \*\*キャラクターカード\*\*

# \- \*\*攻撃カード\*\*

# \- \*\*バフカード（攻撃力UP / HP回復）\*\*

# 

# これらを組み合わせて戦略的にバトルを進めます。

# 

# ---

# 

# \## ゲームルール

# 

# \### 1. バトル開始

# プレイヤーは物理カードを読み取り、各プレイヤーが 1 体ずつキャラクターを選んでフィールドに出します。

# \- Player1 のキャラクター

# \- Player2 のキャラクター

# 

# \### 2. ターン制バトル

# \- プレイヤーは交互にターンを行います。

# \- 1ターンにつき 1 回の行動が可能です。

# 

# \### 3. 行動の種類

# \- \*\*攻撃\*\*

# \- \*\*攻撃力バフ（攻撃力UP）\*\*

# \- \*\*HP回復（回復バフ）\*\*

# 

# \### 4. 攻撃計算式

# ```

# ダメージ ＝（キャラの基礎攻撃力 × 攻撃倍率） × 2（有利属性の場合） − 相手キャラの防御力

# ```

# 

# \### 5. 勝敗条件

# \- どちらかのキャラクターの \*\*HP が 0\*\* になった時点で敗北。

# 

# \## カードデータベース（JSON）仕様

# カードは \*\*カードID（ImageID = DataID）\*\* をキーとして管理されます。

# 

# \### キャラクターカード

# \- キャラ名

# \- HP

# \- 基礎攻撃力

# \- 防御力

# \- 属性

# 

# \### 攻撃カード

# \- 攻撃名

# \- 攻撃倍率

# 

# \### バフカード

# \- バフ名

# \- バフ効果

# &nbsp; - 攻撃力UP：追加攻撃力（基礎攻撃力に加算）

# &nbsp; - HP回復：回復量

# 

# ---

# 

# \## JSON データ例（サンプル）

# ```json

# {

# &nbsp; "001": {

# &nbsp;   "name": "Fire Dragon",

# &nbsp;   "hp": 120,

# &nbsp;   "defense": 10,

# &nbsp;   "element": "fire"

# &nbsp; },

# &nbsp; "101": {

# &nbsp;   "type": "attack",

# &nbsp;   "name": "Flame Burst",

# &nbsp;   "multiplier": 1.5

# &nbsp; },

# &nbsp; "201": {

# &nbsp;   "type": "buff",

# &nbsp;   "name": "Power Up",

# &nbsp;   "effect": {

# &nbsp;     "attackBoost": 15

# &nbsp;   }

# &nbsp; }

# }

# ```

# \## 開発者向けメモ

# \- 各カードデータは DataID に対応する JSON を参照して読み込まれます。

# \- ARTrackedImageManager により画像認識 → データロード → モデル生成が行われます。

# \- バトル進行は BattleManager により制御されます。
